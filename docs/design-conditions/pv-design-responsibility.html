在壓力容器設計中，「簽名」不是形式，而是責任的承擔。

核心問題

在任何設計案中，必須先回答：

  • 誰是設計者（Design Engineer）？
  • 誰對設計結果負責？
  • 誰有權簽署設計文件？

誰可以簽？

只有在以下條件成立時，設計者才應簽署文件：

1. 已定義完整設計輸入（UDR）

  • 壓力、溫度、流體
  • 操作條件與設計條件
  • 法規需求（ASME / PED）
  • 壓力容器使用者設計需求 (UDR) 指南: 從「人事時地物」看 ASME 規範要點
    壓力容器屬於危險性設備，其設計源頭來自於 「使用者設計需求 (User's Design Requirements, UDR)」。一份完整的 UDR 是確保設備安全與符合法規的基礎。

2. 掌握設計方法

  • 規範計算（Code Design）
  • 或工程分析（FEM）

→ 強度計算 → FEM 分析

3. 控制設計輸出

  • 圖面
  • 計算書
  • 材料選用

→ 設計文件

4. 了解製造條件

  • 焊接方式與效率
  • 成形與熱處理
  • 檢驗方法

若設計者未參與上述任一關鍵環節， 則不應簽署設計文件。

什麼是「設計控制權」？

設計控制權（Design Control）指的是：

對設計關鍵變數具有「決定權與修正權」。

包含以下能力：

  • 可修改設計條件（UDR）
  • 可調整材料與厚度
  • 可決定焊接形式與效率
  • 可要求製造變更

不具設計控制權的情境：

  • 僅收到既有圖面
  • 無法變更材料或尺寸
  • 無法介入製造流程
  • 僅被要求「出計算書」

沒有設計控制權的設計者， 本質上只是文件提供者，而非設計負責人。

法律風險來源

以下情境屬於高風險：

1. 代簽設計文件

  • 未參與設計
  • 僅依他人資料簽名

風險：

  • 承擔全部設計責任
  • 無法證明設計合理性

2. 設計與製造分離

  • 設計者無法控制製造
  • 製造偏差未回饋設計

風險：

  • 實際設備不符合設計假設

3. 不完整設計輸入（UDR 缺失）

  • 操作條件不明
  • 設計條件錯誤

風險：

  • 計算結果無效

4. 文件用途錯誤

  • 將分析報告當作認證文件
  • 將估算圖面用於製造

→ 文件分類

工程判斷準則

在簽署前，請確認：

  1. 我是否定義或確認了 UDR？
  2. 我是否能控制設計變更？
  3. 我是否了解製造條件？
  4. 文件是否將用於法規認證？

若以上任一答案為「否」，

則應重新定義責任，而不是直接簽名。

快速判斷（實務）

可以簽：

  • 完整設計案（含 UDR + 設計 + 製造配合）

需要註明責任範圍：

  • 僅分析（FEM / 局部計算）

不應簽：

  • 舊圖補計算
  • 製造後補文件
  • 無設計控制權案件
