1. 🧭 節點定位

Hydrotest 不是用來證明設備「安全」， 而是驗證設備「在特定條件下未發生破壞」。

→ 屬於：Verification（驗證節點）

2. 🔷 ROOT：是否需要進行 Hydrotest？

→ ASME Section VIII Div.1 預設要求（UG-99）

→ 執行 Hydrotest → 替代方案（例：氣壓試驗）

3. 🔶 Hydrotest 執行條件

→ UG-99

3.1. 試驗壓力（Test Pressure）

  • 約 1.3 × MAWP（依 UG-99 調整）
  • 需考慮材料溫度修正

3.2. 試驗介質

  • 水（不可壓縮）
  • 降低儲能風險

3.3. 檢驗內容

  • 是否滲漏（leakage）
  • 是否變形（distortion）
  • 是否破壞（failure）

3.4. 見證角色

→ AI（Authorized Inspector）

  • 見證試驗
  • 確認程序符合 Code

4. 🔶 替代方案（Pneumatic Test）

→ UG-100

4.1. 使用條件

  • 無法注水（重量 / 污染 / 結構限制）
  • 需額外風險控制

4.2. ⚠️ 風險（極高）

  • 壓縮氣體儲能高
  • 破裂 → 爆炸效應

5. 🔥 關鍵工程判斷（Critical Thinking）

5.1. Hydrotest 是否驗證設計正確？

答案：否

→ 它只驗證： 「設備在試驗壓力下沒有失效」

5.2. Hydrotest 是否代表設備安全？

答案：否

→ 無法涵蓋：

  • 疲勞（fatigue）
  • 局部應力集中
  • 長期腐蝕
  • 熱應力

5.3. Hydrotest 真正驗證的是什麼？

  • 製造品質（焊接 / 組裝）
  • 結構完整性（短期）

6. 🔷 常見工程誤解（非常重要）

❌ Hydrotest 過 = 設計沒問題 ❌ Hydrotest 過 = 可以安全運行 ❌ Hydrotest 是設計驗證

✔ 正確理解：

Hydrotest 是「最低限度的製造驗證」

7. 🔗 與其他節點關聯

  • 厚度設計
  • 焊接效率

8. 🧭 收斂（首頁可引用）

Hydrotest 不是安全的證明， 而是「尚未失敗的證明」。
