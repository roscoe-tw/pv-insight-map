UG-99 壓力試驗（Hydrostatic Test）

本節為壓力容器最終完整性驗證節點， 涵蓋試驗壓力決定、試驗方式選擇、結構風險與責任邊界。

通過試驗 ≠ 設計正確 試驗的本質是「驗證」，不是「設計替代」

試驗方式選擇（Test Method）

是否可進行水壓試驗？

  • → YES → UG-99（Hydrostatic Test）
  • → NO → UG-100（Pneumatic Test）

為何不能用水？

  • 污染風險（Process sensitive）
  • 重量限制（大型設備）
  • 無法乾燥（如低溫設備）

風險判斷（氣壓）

  • 是否完成風險評估？
  • 是否設置安全距離？
  • 是否分段升壓？

試驗壓力決定（Test Pressure）

計算公式

P[test] = 1.3 × MAWP × (S[t] / S[d])

必要輸入

  • MAWP
  • 設計溫度（Design Temp）
  • 試驗溫度（Test Temp）
  • 材料 Allowable Stress（St, Sd）

判斷節點

St / Sd 是否 > 1？

  • → YES → 試驗壓力提高（高風險）
  • → NO → 正常

是否造成過度應力（Overstress）？

  • Nozzle
  • Flange
  • Support（Skirt / Saddle）

試驗條件檢查（Pre-Test Check）

壓力表

  • 是否校驗？
  • 量程是否合理（1.5~4 × Test Pressure）？

排氣

  • 是否完全排除 air pocket？

支撐條件

  • 是否與設計條件不同？
  • 是否產生額外應力？

升壓與保壓（Pressurization）

升壓方式

  • 是否緩慢升壓？
  • 是否分段檢查？

保壓

  • 是否達到穩定？
  • 是否觀察壓力下降？

檢驗判定（Acceptance Criteria）

是否有洩漏？

  • → YES → 不合格
  • → NO → 合格

是否有可見變形？

  • Bulging
  • Permanent deformation

是否有異常聲音 / 局部失效？

結構風險評估（Critical Risk）

最弱點分析

  • Nozzle reinforcement（UG-37）
  • Manway
  • Flange
  • Weld HAZ

試驗載荷 vs 設計載荷

  • 是否存在：
    • 額外外力
    • 支撐拘束
    • Nozzle load 放大？

責任邊界（Responsibility Boundary）

誰決定 Test Pressure？

  • 設計工程師？
  • 製造廠？

誰確認材料性質（St / Sd）？

誰負責試驗程序？

  • 是否有書面 Procedure？

AI（Authorized Inspector）角色

  • 見證（Witness）
  • 不負設計責任

UG-99 是責任交會點： 設計 / 製造 / 檢驗 在此匯合

常見工程錯誤（Common Pitfalls）

  • 直接使用 1.3 × MAWP（忽略 St / Sd）
  • 未檢查 nozzle / flange 在 test condition
  • 將「通過試驗」視為設計正確
  • 設計工程師未參與試驗條件設定

工程建議（Engineering Actions）

試驗前

  • Nozzle load 評估（WRC / FEA）
  • Flange rating check（Test Temp）
  • Support stress 檢查

文件化

  • Test Procedure
  • 壓力計校驗紀錄
  • 試驗報告（含壓力曲線）
